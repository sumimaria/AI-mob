What is Reinforcement Learning?
Agents, environments, rewards and the goal of maximising return
The Core Idea

An agent interacts with an environment. It takes actions, observes new states, and receives rewards. Its goal: learn a policy that maximises total reward over time — without being told the rules.

Key Concepts

State (s): What the agent observes. Action (a): What the agent does. Reward (r): Feedback signal. Policy (π): Mapping from state to action. Return (G): Cumulative future reward.

vs Supervised Learning

In supervised ML, you provide labelled answers. In RL, there are no labels — only sparse reward signals. The agent must explore to discover what works and exploit what it knows.

Real-World Uses

Game AI (AlphaGo, OpenAI Five), robot locomotion, trading strategies, recommendation systems, autonomous driving, drug discovery, RLHF for training ChatGPT.

02🗺️
Markov Decision Process (MDP)
The mathematical foundation of RL — states, transitions, rewards and discount factor

🗺️ What is an MDP? Foundation

An MDP is a formal framework for sequential decision-making. It is defined by a tuple (S, A, P, R, γ) — states, actions, transition probabilities, reward function, and discount factor.

  • Markov Property: "The future depends only on the present, not the past." The next state depends only on the current state and action
  • Discount factor γ: Controls how much the agent values future vs immediate rewards. γ=0 → greedy; γ=1 → far-sighted
  • Return Gₜ = rₜ + γrₜ₊₁ + γ²rₜ₊₂ + … — sum of discounted future rewards
  • Value function V(s): Expected return from state s following policy π

Python — MDP Value Iteration
import numpy as np

# 4×4 Grid World MDP
# States: 0-15, Goal: 15 (bottom-right), Hole: 5
n_states, n_actions = 16, 4  # UP DOWN LEFT RIGHT
gamma = 0.9
V = np.zeros(n_states)
R = np.full(n_states, -1.0)  # step cost
R[15] = +10.0               # goal reward
R[5]  = -10.0               # hole penalty

# Value Iteration
for _ in range(100):
    V_new = np.copy(V)
    for s in range(n_states):
        if s in [5, 15]: continue   # terminal states
        row, col = s//4, s%4
        next_states = {
            0: 4*(row-1)+col if row>0 else s,  # UP
            1: 4*(row+1)+col if row<3 else s,  # DOWN
            2: 4*row+(col-1) if col>0 else s,  # LEFT
            3: 4*row+(col+1) if col<3 else s,  # RIGHT
        }
        V_new[s] = max(R[ns] + gamma * V[ns]
                       for ns in next_states.values())
    V = V_new
print(V.reshape(4,4).round(2))
03📊
Q-Learning & The Q-Table
Model-free TD learning — building a table of action values to guide the agent

📊 Q-Learning Model-Free

Q-Learning learns a Q-table: Q(s, a) = expected return if you take action a in state s and follow the optimal policy afterwards. The Bellman update equation is applied after each step until the table converges.

  • Bellman Equation: Q(s,a) ← Q(s,a) + α[r + γ·max Q(s',a') − Q(s,a)]
  • α (learning rate): How quickly new information overwrites old
  • ε-greedy: Explore randomly with probability ε, exploit best action otherwise
  • Convergence: Guaranteed to converge to optimal Q* given sufficient exploration

Python — Q-Learning from Scratch
import numpy as np
import random

# Environment: 5×5 maze, start=(0,0), goal=(4,4)
ROWS, COLS = 5, 5
ACTIONS = [(-1,0),(1,0),(0,-1),(0,1)]  # UP DOWN LEFT RIGHT
WALLS = {(1,1),(2,1),(2,3),(3,2)}

# Q-table: shape (rows, cols, n_actions)
Q = np.zeros((ROWS, COLS, 4))
alpha, gamma, eps = 0.5, 0.9, 0.2

def step(s, a):
    nr, nc = s[0]+ACTIONS[a][0], s[1]+ACTIONS[a][1]
    if not (0<=nrand 0<=ncor (nr,nc) in WALLS:
        return s, -1, False   # hit wall
    done = (nr==ROWS-1 and nc==COLS-1)
    return (nr,nc), +10 if done else -1, done

for ep in range(1000):
    s = (0, 0)
    while True:
        # ε-greedy action selection
        a = random.randint(0,3) if random.random()else Q[s].argmax()
        ns, r, done = step(s, a)
        # Bellman update
        Q[s][a] += alpha*(r + gamma*Q[ns].max() - Q[s][a])
        s = ns
        if done: break
print("Trained! Optimal actions:", Q.argmax(axis=2))

04🧠
Deep Q-Network (DQN)
Replacing the Q-table with a neural network — scaling RL to Atari and beyond

🧠 DQN Deep RL

Q-tables become impossible for large state spaces (e.g. pixels on a screen). DQN replaces the table with a neural network that approximates Q(s,a). DeepMind's 2013 breakthrough — one DQN played 49 Atari games at human level.

  • Experience Replay: Stores past transitions in a buffer; samples random mini-batches to break correlations
  • Target Network: A frozen copy of the Q-network, updated every N steps — stabilises training
  • Input: Raw pixels (84×84 grayscale frames) stacked 4 deep to encode motion
  • Loss: MSE between predicted Q(s,a) and target r + γ·max Q(s',a')

Python — DQN with PyTorch
import torch
import torch.nn as nn
import numpy as np
from collections import deque
import random

# Q-Network architecture
class DQN(nn.Module):
    def __init__(self, state_dim, action_dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim, 128), nn.ReLU(),
            nn.Linear(128, 64),   nn.ReLU(),
            nn.Linear(64, action_dim)
        )
    def forward(self, x): return self.net(x)

# Replay Buffer
buffer = deque(maxlen=10000)

# Main DQN training loop
q_net     = DQN(state_dim=4, action_dim=2)
target    = DQN(state_dim=4, action_dim=2)
target.load_state_dict(q_net.state_dict())  # sync
optimizer = torch.optim.Adam(q_net.parameters(), lr=1e-3)
criterion = nn.MSELoss()

for step in range(100000):
    # Sample mini-batch from replay buffer
    batch = random.sample(buffer, 64)
    s, a, r, ns, done = zip(*batch)
    s  = torch.FloatTensor(s)
    ns = torch.FloatTensor(ns)
    # Compute TD target using frozen target network
    with torch.no_grad():
        target_q = r + 0.99*target(ns).max(1).values*(~done)
    # MSE loss between predicted and target Q
    loss = criterion(q_net(s).gather(1,a), target_q)
    optimizer.zero_grad(); loss.backward(); optimizer.step()
    if step % 1000 == 0:
        target.load_state_dict(q_net.state_dict())  # sync target

Policy Gradient Methods
Directly optimising the policy — REINFORCE, Actor-Critic and PPO

🎯 Policy Gradient Direct Optimisation

Instead of learning Q-values, policy gradient methods directly parameterise and optimise the policy π(a|s; θ). The gradient ∇J(θ) points in the direction that increases expected return.

  • REINFORCE: Monte-Carlo policy gradient — update after full episode using actual returns
  • Actor-Critic (A2C/A3C): Actor outputs policy, Critic estimates value — reduces variance
  • PPO: Proximal Policy Optimisation — most widely used today (OpenAI Five, ChatGPT RLHF)
  • Advantages over Q-learning: Works in continuous action spaces; can learn stochastic policies

Exploration vs Exploitation & Multi-Armed Bandits
The fundamental dilemma — trying new things vs using what already works

🎰 The Exploration-Exploitation Dilemma Bandits

Should the agent keep doing what worked before (exploit) or try new actions that might be better (explore)? The Multi-Armed Bandit problem is the simplest RL setting — no state transitions, just choose which arm to pull each round to maximise reward.

  • ε-greedy: Explore with probability ε, exploit best known arm otherwise
  • UCB (Upper Confidence Bound): Pull the arm with highest optimistic estimate — prefer uncertain arms
  • Thompson Sampling: Bayesian approach — sample from reward distribution belief
  • Regret: Difference between optimal reward and actual reward collected — minimise this

Python — Multi-Armed Bandit (ε-greedy vs UCB)
import numpy as np

n_arms, n_rounds = 5, 1000
# True hidden payout probabilities (unknown to agent)
true_means = np.random.uniform(0, 1, n_arms)

def epsilon_greedy(eps=0.1):
    Q = np.zeros(n_arms)   # estimated means
    N = np.zeros(n_arms)   # pull counts
    rewards = []
    for t in range(n_rounds):
        a = np.random.randint(n_arms) if np.random.random() < eps \
            else Q.argmax()
        r = np.random.normal(true_means[a], 0.1)
        N[a] += 1
        Q[a] += (r - Q[a]) / N[a]   # incremental mean update
        rewards.append(r)
    return rewards

def ucb(c=2):
    Q = np.zeros(n_arms);  N = np.zeros(n_arms)
    rewards = []
    for t in range(n_rounds):
        # UCB score: estimated mean + exploration bonus
        ucb_score = Q + c*np.sqrt(np.log(t+1)/(N+1e-5))
        a = ucb_score.argmax()
        r = np.random.normal(true_means[a], 0.1)
        N[a] += 1;  Q[a] += (r-Q[a])/N[a]
        rewards.append(r)
    return rewards
