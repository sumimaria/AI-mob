What is Computer Vision?
Teaching machines to see, interpret and reason about visual information
What Machines See

A computer sees an image as a grid of numbers — each pixel is one or three values (0-255) representing brightness or RGB colour. CV teaches algorithms to extract meaning from these grids.

The CV Pipeline

Image → Preprocess → Feature Extract (CNN filters) → Classify / Detect / Segment → Output. Each stage transforms raw pixels into higher-level understanding.

Core Tasks

Classification (what is this?), Detection (where are things?), Segmentation (which pixel belongs where?), Face Recognition, Pose Estimation, Depth Estimation.

Real-World Uses

Self-driving cars, medical imaging (tumour detection), face unlock, quality control in factories, satellite image analysis, augmented reality, sports analytics.

Image Basics & Pixels
How digital images are stored — RGB channels, grayscale, and colour spaces

🎨 Pixels & Colour Spaces Foundation

Every digital image is a rectangular grid of pixels. Each pixel stores colour as three values: Red, Green, Blue (0-255). CV algorithms often convert images to other colour spaces — grayscale (1 channel), HSV (hue-saturation-value), or LAB — because different tasks benefit from different representations.

  • Grayscale: 1 channel — brightness only. Simplifies many algorithms
  • HSV: Separates colour (hue) from intensity — great for colour-based segmentation
  • Normalisation: Scale pixel values to [0,1] or [-1,1] before feeding to neural networks

03🔎
Image Classification
Assigning a label to an entire image — the "what is this?" task

🔎 Image Classification CNN — Convolutional Neural Network

Classification assigns a single label to an entire image. A CNN (Convolutional Neural Network) passes the image through convolutional layers to extract features, then a fully connected layer outputs probabilities for each class via softmax. The class with highest probability is the prediction.

  • AlexNet (2012): First deep CNN to win ImageNet — sparked the DL revolution
  • VGG / ResNet: Deeper networks with skip connections to avoid vanishing gradients
  • Transfer Learning: Start from ImageNet weights, fine-tune on your data
  • ImageNet: 1000-class benchmark — 1.2M training images

Python — CNN Image Classifier (Keras/TensorFlow)
from tensorflow.keras.applications import ResNet50
from tensorflow.keras.applications.resnet50 import preprocess_input, decode_predictions
from tensorflow.keras.preprocessing import image
import numpy as np

# Load pre-trained ResNet50 (ImageNet weights)
model = ResNet50(weights='imagenet')

# Load and preprocess your image
img = image.load_img('cat.jpg', target_size=(224, 224))
x   = image.img_to_array(img)
x   = np.expand_dims(x, axis=0)    # shape: (1, 224, 224, 3)
x   = preprocess_input(x)            # normalise for ResNet

# Predict (outputs 1000-class probabilities)
preds = model.predict(x)
top5  = decode_predictions(preds, top=5)[0]

for rank, (id_, name, prob) in enumerate(top5, 1):
    print(f"{rank}. {name:20} {prob:.2%}")
# 1. tabby_cat           94.72%
# 2. Egyptian_cat         3.81%

# Transfer learning: fine-tune on your own classes
from tensorflow.keras import layers, models
base = ResNet50(weights='imagenet', include_top=False)
base.trainable = False  # freeze backbone
model = models.Sequential([base, layers.GlobalAveragePooling2D(),
    layers.Dense(5, activation='softmax')])  # 5 custom classes

04📦
Object Detection
Finding AND locating multiple objects in an image with bounding boxes

📦 Object Detection Localization

Detection goes beyond classification — it must answer where each object is, not just what. The output is a set of bounding boxes + class labels + confidence scores.

  • YOLO (You Only Look Once): Single-pass detector — extremely fast, real-time
  • Faster R-CNN: Region proposal network + classifier — very accurate
  • IoU (Intersection over Union): Measures overlap between predicted and ground-truth box
  • NMS (Non-Max Suppression): Removes duplicate detections for the same object
  • mAP: Mean Average Precision — the standard detection benchmark metric

Python — Object Detection with YOLOv8
from ultralytics import YOLO
import cv2

# Load pre-trained YOLOv8 nano model
model = YOLO('yolov8n.pt')

# Detect objects in an image
results = model('street.jpg', conf=0.5)

# Parse results
for r in results:
    boxes = r.boxes
    for box in boxes:
        x1, y1, x2, y2 = box.xyxy[0].tolist()
        cls_id  = int(box.cls[0])
        conf    = float(box.conf[0])
        name    = model.names[cls_id]
        print(f"{name}: {conf:.0%}  [{x1:.0f},{y1:.0f},{x2:.0f},{y2:.0f}]")

# Draw and save annotated image
annotated = results[0].plot()
cv2.imwrite('annotated.jpg', annotated)

# Real-time webcam detection
cap = cv2.VideoCapture(0)
while cap.isOpened():
    ret, frame = cap.read()
    results    = model(frame)
    cv2.imshow('YOLO', results[0].plot())

Image Segmentation
Pixel-level understanding — semantic, instance and panoptic segmentation

🎭 Segmentation Pixel-Level

Segmentation assigns a label to every pixel in an image — the finest level of spatial understanding. Different from detection which only draws bounding boxes.

  • Semantic: Each pixel gets a class label (all cars = one colour). No distinction between instances
  • Instance: Each object instance gets a unique mask (car #1 vs car #2)
  • Panoptic: Combines semantic + instance — best of both worlds
  • U-Net: Encoder-decoder architecture with skip connections — dominates medical imaging
  • SAM (Segment Anything): Meta's foundation model — click any object, get a mask

Python — Semantic Segmentation (SegFormer)
from transformers import SegformerForSemanticSegmentation, SegformerImageProcessor
from PIL import Image
import torch, numpy as np

# Load SegFormer fine-tuned on ADE20K (150 classes)
processor = SegformerImageProcessor.from_pretrained(
    "nvidia/segformer-b0-finetuned-ade-512-512")
model = SegformerForSemanticSegmentation.from_pretrained(
    "nvidia/segformer-b0-finetuned-ade-512-512")

image = Image.open("street.jpg")
inputs = processor(images=image, return_tensors="pt")

with torch.no_grad():
    outputs = model(**inputs)

# Upsample logits to original image size
logits = outputs.logits                   # shape (1, num_classes, H/4, W/4)
upsampled = torch.nn.functional.interpolate(
    logits, size=image.size[::-1], mode="bilinear")

# Get per-pixel class prediction
seg_map = upsampled.argmax(dim=1)[0].numpy()
# seg_map[y, x] = class index for pixel (x, y)

# U-Net for medical image segmentation
from segmentation_models_pytorch import Unet
unet = Unet(encoder_name="resnet34",
             encoder_weights="imagenet",
             in_channels=1, classes=1)  # grayscale → binary mask
06😀
Face Detection & Recognition
Locating faces and verifying identity — from Haar cascades to ArcFace

😀 Face Detection & Recognition Biometrics

Face detection finds where faces are. Face recognition identifies who a face belongs to by comparing face embeddings — 128-512 dimensional vectors that encode the unique geometry of a face.

  • Haar Cascade: Fast, classical method — slides a feature detector across image regions
  • MTCNN: Multi-task CNN — simultaneously detects face, eyes, nose and mouth landmarks
  • FaceNet / ArcFace: Deep CNN that maps each face to an embedding vector
  • Verification vs Identification: 1:1 (is this person X?) vs 1:N (who is this person?)

Python — Face Recognition (face_recognition library)
import face_recognition
import numpy as np
import cv2

# Load known face and compute its 128-d embedding
known_img  = face_recognition.load_image_file("person_a.jpg")
known_enc  = face_recognition.face_encodings(known_img)[0]

# Load unknown face
unknown_img = face_recognition.load_image_file("unknown.jpg")
unknown_enc = face_recognition.face_encodings(unknown_img)[0]

# Compute Euclidean distance (threshold typically 0.6)
distance = np.linalg.norm(known_enc - unknown_enc)
match    = distance < 0.6

print(f"Distance: {distance:.3f}")
print("MATCH ✓" if match else "NO MATCH ✗")

# Detect all faces in a group photo
group_img  = face_recognition.load_image_file("group.jpg")
locations  = face_recognition.face_locations(group_img)
encodings  = face_recognition.face_encodings(group_img, locations)
print(f"Found {len(locations)} faces")

Edge Detection & Convolution Filters
Sobel, Canny, Laplacian — how CNNs learn to detect boundaries

⚡ Edge Detection Filters

Edges mark the boundaries between objects. Detecting them is the first step in many CV pipelines. Convolutional filters (kernels) slide over images and highlight regions of rapid intensity change.

  • Sobel: Gradient in X or Y direction — detects horizontal or vertical edges
  • Canny: Multi-stage: Gaussian blur → Sobel → Non-max suppression → Hysteresis thresholding
  • Laplacian: Second derivative — detects all edges regardless of orientation
  • CNN kernels: Learned automatically — hundreds of task-specific filters

gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

# Sobel — gradient in X and Y
sobel_x = cv2.Sobel(gray, cv2.CV_64F, 1, 0, ksize=3)
sobel_y = cv2.Sobel(gray, cv2.CV_64F, 0, 1, ksize=3)
sobel   = np.sqrt(sobel_x**2 + sobel_y**2)
