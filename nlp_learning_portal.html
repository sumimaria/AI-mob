What is NLP?
The bridge between human language and machine understanding
The Challenge

Human language is ambiguous, contextual and constantly evolving. "Bank" can mean a riverbank or a financial institution. Machines must learn these nuances from data.

The Pipeline

Raw text → Tokenise → Clean → Vectorise → Model → Output. Each step transforms unstructured text into something a machine can reason about.

Key Applications

Chatbots, search engines, spam filters, translation, autocomplete, voice assistants, medical record analysis, legal document review.

Evolution

Rule-based (1950s) → Statistical (1990s) → Machine Learning (2000s) → Deep Learning & Transformers (2017+) → Large Language Models (2020+).

Tokenisation & Text Preprocessing
Breaking text into pieces — the essential first step of every NLP pipeline

✂️ What is Tokenisation? Foundation

Tokenisation splits raw text into individual units (tokens) — words, subwords, or characters. Without it, a model sees an unbreakable string of characters. Most modern LLMs use subword tokenisation (BPE, WordPiece) to handle unknown words efficiently.

  • Word tokenise: Split on spaces/punctuation → ["Hello", ",", "world", "!"]
  • Subword (BPE): "unhappiness" → ["un", "happiness"] — handles rare words
  • Stop words: Common words (the, is, at) often removed to reduce noise
  • Stemming/Lemmatisation: "running" → "run" — normalise word forms

Python — Tokenisation with NLTK
import nltk
from nltk.tokenize import word_tokenize, sent_tokenize
from nltk.corpus import stopwords
from nltk.stem import PorterStemmer, WordNetLemmatizer

text = "The quick brown fox jumps over the lazy dog!"

# Word tokenisation
tokens = word_tokenize(text)
print(tokens)
# ['The', 'quick', 'brown', 'fox', 'jumps', 'over', 'the', 'lazy', 'dog', '!']

# Remove stop words
stop_words = set(stopwords.words('english'))
filtered = [w for w in tokens if w.lower() not in stop_words]
print(filtered)  # ['quick', 'brown', 'fox', 'jumps', 'lazy', 'dog', '!']

# Stemming
stemmer = PorterStemmer()
stems = [stemmer.stem(w) for w in filtered]
print(stems)  # ['quick', 'brown', 'fox', 'jump', 'lazi', 'dog', '!']

# Subword tokenisation with HuggingFace
from transformers import AutoTokenizer
tokenizer = AutoTokenizer.from_pretrained('bert-base-uncased')
subwords = tokenizer.tokenize("unhappiness")
print(subwords)  # ['un', '##happiness']

Sentiment Analysis
Teaching machines to understand emotion and opinion in text

😊 What is Sentiment Analysis? Classification

Sentiment analysis classifies text as positive, negative, or neutral. Used in product reviews, social media monitoring, customer feedback, and stock market sentiment. Modern models also detect fine-grained emotions: joy, anger, fear, surprise.

  • Lexicon-based: Dictionary of positive/negative words + score
  • ML-based: Train classifier (Naive Bayes, SVM) on labelled reviews
  • Deep learning: BERT fine-tuned on sentiment datasets (SST-2, IMDB)
  • Aspect-based: "The camera is great but battery life is terrible" → mixed

# Method 1: Rule-based VADER (fast, no training needed)
from nltk.sentiment import SentimentIntensityAnalyzer

sia = SentimentIntensityAnalyzer()
text = "This product is absolutely amazing! Highly recommend!"
scores = sia.polarity_scores(text)
print(scores)
# {'neg': 0.0, 'neu': 0.306, 'pos': 0.694, 'compound': 0.8316}

verdict = "Positive" if scores['compound'] > 0.05 else \
          "Negative" if scores['compound'] < -0.05 else "Neutral"

# Method 2: Deep Learning with HuggingFace Transformers
from transformers import pipeline

sentiment_pipe = pipeline("sentiment-analysis",
    model="distilbert-base-uncased-finetuned-sst-2-english")

result = sentiment_pipe(text)[0]
print(result)
# {'label': 'POSITIVE', 'score': 0.9998}

# Batch analysis of multiple reviews
reviews = ["Great product!", "Terrible experience", "It was okay"]
results = sentiment_pipe(reviews)
for r, res in zip(reviews, results):
    print(f"{r[:30]:30} → {res['label']} ({res['score']:.2%})")

04🏷️
Named Entity Recognition (NER)
Identifying real-world objects in text — people, places, organisations, dates

🏷️ What is NER? Information Extraction

NER locates and classifies named entities in text into predefined categories. A news article about "Elon Musk founded SpaceX in Hawthorne, California in 2002 with $100M" contains persons, organisations, locations, dates and monetary values.

  • PERSON: People's names — "Barack Obama", "Marie Curie"
  • ORG: Companies, institutions — "Google", "Harvard University"
  • LOC / GPE: Locations — "Paris", "Amazon River"
  • DATE / TIME: Temporal expressions — "last Monday", "1969"
  • MONEY / PERCENT: Quantities — "$5 billion", "12%"

Python — NER with spaCy
import spacy

# Load the pre-trained English model
nlp = spacy.load("en_core_web_sm")  # pip install spacy

text = """Elon Musk founded SpaceX in Hawthorne, California in 2002
         with an investment of $100 million."""

doc = nlp(text)

# Print all named entities
for ent in doc.ents:
    print(f"{ent.text:20} → {ent.label_:10} ({spacy.explain(ent.label_)})")

# Output:
# Elon Musk            → PERSON     (People, fictional chars)
# SpaceX               → ORG        (Companies, agencies)
# Hawthorne            → GPE        (Countries, cities, states)
# California           → GPE        (Countries, cities, states)
# 2002                 → DATE       (Absolute or relative dates)
# $100 million         → MONEY      (Monetary values)

# Visualise in browser
from spacy import displacy
displacy.serve(doc, style="ent")

Text Classification
Categorising text automatically — spam detection, topic labelling and more

🗂️ What is Text Classification? Supervised

Text classification assigns predefined labels or categories to a piece of text. The model learns from labelled training examples which patterns correspond to which category.

  • Binary: Spam/Ham, Positive/Negative
  • Multi-class: News → Sports / Politics / Tech / Entertainment
  • Multi-label: A document can belong to multiple categories simultaneously
  • Zero-shot: Classify into categories the model has never seen during training

Python — Naive Bayes Spam Classifier
from sklearn.naive_bayes import MultinomialNB
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.pipeline import Pipeline
from sklearn.model_selection import train_test_split
from sklearn.metrics import classification_report
import pandas as pd

# Load SMS Spam dataset
df = pd.read_csv('spam.csv', encoding='latin-1')[['v1','v2']]
df.columns = ['label', 'text']

X_train, X_test, y_train, y_test = train_test_split(
    df['text'], df['label'], test_size=0.2, random_state=42)

# Pipeline: TF-IDF → Naive Bayes
pipe = Pipeline([
    ('tfidf', TfidfVectorizer(stop_words='english', max_features=5000)),
    ('clf',   MultinomialNB(alpha=0.1))
])
pipe.fit(X_train, y_train)

# Evaluate
y_pred = pipe.predict(X_test)
print(classification_report(y_test, y_pred))
# Accuracy: ~98.5%

# Predict new email
new_email = ["WIN a FREE iPhone! Click NOW! Limited offer!!!"]
print(pipe.predict(new_email))  # ['spam']

Machine Translation
How computers translate between human languages — from rule-based to Neural MT

🌍 Machine Translation Seq2Seq

Machine Translation converts text from one language to another automatically. Modern systems use Neural Machine Translation (NMT) — an encoder-decoder Transformer that encodes the source sentence into a context vector, then decodes it token-by-token in the target language.

  • Rule-based (RBMT): Hand-crafted grammar rules — rigid, expensive to build
  • Statistical (SMT): Learns phrase mappings from bilingual corpora (IBM Model, Moses)
  • Neural (NMT): End-to-end Seq2Seq with attention — Google Translate, DeepL
  • Attention: Each output word looks back at relevant input words

Text Summarisation
Extractive vs Abstractive — condensing documents while preserving meaning

📝 Text Summarisation Generation

Summarisation compresses a long document into a shorter version while preserving key information. Two main approaches:

  • Extractive: Selects and returns actual sentences from the original — no new text generated. Uses TF-IDF scores, TextRank (graph-based).
  • Abstractive: Generates entirely new sentences that paraphrase the content. Uses seq2seq Transformers (BART, T5, Pegasus).
  • ROUGE score: Measures summarisation quality (recall of n-grams vs reference)

Python — Extractive Summarisation (TextRank)
import numpy as np
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity
import networkx as nx
from nltk.tokenize import sent_tokenize

def textrank_summary(text, n=2):
    sentences = sent_tokenize(text)
    
    # Build TF-IDF similarity matrix
    tfidf = TfidfVectorizer()
    tfidf_matrix = tfidf.fit_transform(sentences)
    sim_matrix = cosine_similarity(tfidf_matrix)
    
    # PageRank on sentence graph
    graph = nx.from_numpy_array(sim_matrix)
    scores = nx.pagerank(graph)
    
    # Select top-n sentences in original order
    ranked = sorted(scores, key=scores.get, reverse=True)[:n]
    ranked_sorted = sorted(ranked)  # restore order
    
    return ' '.join([sentences[i] for i in ranked_sorted])

text = """Artificial intelligence is transforming industries..."""
summary = textrank_summary(text, n=2)
print(summary)

Python — Abstractive Summarisation (BART)
from transformers import pipeline

# BART fine-tuned on CNN/DailyMail news summarisation
summarizer = pipeline("summarization",
    model="facebook/bart-large-cnn")

text = """The smartphone market has seen dramatic changes over
the past decade. Apple continues to dominate premium segments
while Samsung leads in overall volume..."""

summary = summarizer(text,
    max_length=60,       # max output tokens
    min_length=20,       # min output tokens
    do_sample=False)[0]  # greedy decoding

print(summary['summary_text'])
# "Apple dominates premium smartphones while Samsung leads
#  volume. Chinese brands capture mid-range markets as 5G
#  drives upgrade cycles and foldables gain accessibility."

# Alternative: T5 model
t5_summarizer = pipeline("summarization", model="t5-small")
# Prefix "summarize: " tells T5 the task
result = t5_summarizer("summarize: " + text)[0]

Question Answering
Machines that read documents and answer questions — the basis of search and AI assistants

❓ Extractive QA Reading Comprehension

Extractive QA finds the exact span in a passage that answers a question. BERT-based models predict the start and end token positions of the answer. This powers Google's featured snippets and enterprise knowledge bases.

  • SQuAD: The benchmark dataset — 100k question-passage-answer triples
  • BERT/RoBERTa: Fine-tuned to predict answer spans with >90% F1 on SQuAD
  • Open-domain QA: No passage given — model retrieves relevant documents first (RAG)

Python — Extractive QA with BERT
from transformers import pipeline

# Load fine-tuned BERT QA model (trained on SQuAD)
qa_pipe = pipeline("question-answering",
    model="deepset/roberta-base-squad2")

context = """The Amazon rainforest covers over 5.5 million square
kilometres. It is home to 390 billion individual trees across
16,000 species. The Amazon River is the second longest in the world."""

question = "How large is the Amazon rainforest?"

result = qa_pipe(question=question, context=context)
print(f"Answer:     {result['answer']}")
print(f"Confidence: {result['score']:.1%}")
print(f"Start pos:  {result['start']}")
print(f"End pos:    {result['end']}")
# Answer:     5.5 million square kilometres
# Confidence: 96.3%

# Multi-question batch
questions = [
    "How many tree species are in the Amazon?",
    "What percentage of oxygen does it produce?"
]
for q in questions:
    r = qa_pipe(question=q, context=context)
    print(f"Q: {q}\nA: {r['answer']} ({r['score']:.0%})\n")

Word Embeddings
Converting words to numbers — Word2Vec, GloVe and contextual embeddings

🔢 What are Word Embeddings? Representation

Machines can't process raw words — they need numbers. Word embeddings map each word to a dense vector in a high-dimensional space where semantically similar words are geometrically close. The famous example: King − Man + Woman ≈ Queen.

  • Word2Vec: Predicts a word from its neighbours (CBOW) or neighbours from a word (Skip-gram)
  • GloVe: Uses global co-occurrence statistics across the entire corpus
  • FastText: Character n-grams — handles morphology and rare words better
  • Contextual (BERT): Same word gets different vectors in different contexts

Python — Word2Vec with Gensim
import gensim.downloader as api

# Load pre-trained Word2Vec (Google News, 3M words, 300-dim)
model = api.load("word2vec-google-news-300")

# Word similarity
sim = model.similarity('king', 'queen')
print(f"king ↔ queen: {sim:.3f}")   # 0.651

# Most similar words
similar = model.most_similar('computer', topn=5)
for word, score in similar:
    print(f"  {word:15} {score:.3f}")
# computers      0.797
# laptop         0.762
# software       0.721

# Word analogy: king - man + woman = ?
result = model.most_similar(
    positive=['king', 'woman'],
    negative=['man'], topn=1)
print(f"king - man + woman = {result[0][0]}")
# king - man + woman = queen ✓
